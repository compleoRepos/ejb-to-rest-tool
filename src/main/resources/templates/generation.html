Options de generation

Conformite BIAN activee

Les API REST generees suivront le standard BIAN :

  • URLs BIAN : /{service-domain}/{cr-plural}/{cr-id}/{behavior-qualifier}
  • Action Terms : Initiate, Retrieve, Update, Execute, Control, Capture
  • Service Domains : Current Account, Payment Execution, Party Reference Data, etc.
  • Behavior Qualifiers : balances, payments, statements, etc.
  • Un fichier BIAN_MAPPING.md sera genere avec le detail du mapping
Configurer le mapping BIAN Recommande : verifiez le mapping avant de generer

Mode de transport (couche Infrastructure)

Mode REST Adapter active

L'API generee appellera un adapter WebSphere via HTTP au lieu de l'EJB directement :

  • RestAdapter : Appelle l'adapter WebSphere via RestTemplate (profil rest)
  • RestClientConfig : Configuration des timeouts, retry, interceptors
  • Pact Consumer Tests : Contrats generes automatiquement pour l'equipe WebSphere
  • Resilience4j : Circuit Breaker, Retry, Bulkhead sur chaque appel
  • Les profils jndi et mock restent disponibles comme fallback
