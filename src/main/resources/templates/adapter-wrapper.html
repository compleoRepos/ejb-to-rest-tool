Pipeline Adapter Contract → Wrapper BIAN
Uploadez le contrat JSON normalise d'un Adapter WebSphere. L'outil genere automatiquement :
  • Controller BIAN avec enveloppe ApiRequest/ApiResponse
  • Service Interface (couche domaine decouple)
  • RestAdapter avec Circuit Breaker, Retry, Idempotency
  • DTOs Request/Response a partir des champs du contrat
  • Tests Pact (Consumer-Driven Contract)
  • Tests d'integration avec MockAdapter
  • Configuration Spring Boot + Resilience4j

Glissez un fichier JSON ou parcourir

Format accepte : .json (contrat normalise d'adapter)
Editeur JSON en ligne
Collez ou editez directement le contrat JSON de l'adapter. Cliquez sur "Valider" pour verifier la syntaxe, puis sur "Generer" pour lancer la generation du Wrapper BIAN.
Format du contrat JSON normalise
Voici le format que les Adapters WebSphere doivent respecter. Chaque adapter expose un ou plusieurs endpoints qui correspondent a des capacites metier (EJB legacy). 
{
  "adapter_name": "CommandChequier",
  "adapter_base_url": "http://websphere-host:9080/api/adapter",
  "description": "Adapter pour la commande de chequiers",
  "version": "1.0.0",
  "endpoints": [
    {
      "operation": "enrg_commande",
      "method": "POST",
      "path": "/command-chequier/enrg-commande",
      "summary": "Enregistrer une commande de chequier",
      "idempotent": true,
      "timeout_seconds": 30,
      "max_retries": 3,
      "request_fields": [
        { "name": "numeroCarte", "type": "String", "required": true, "description": "Numero de carte bancaire" },
        { "name": "quantite", "type": "Integer", "required": true, "description": "Nombre de chequiers" },
        { "name": "adresseLivraison", "type": "String", "required": false, "description": "Adresse de livraison" }
      ],
      "response_fields": [
        { "name": "code", "type": "String", "description": "Code retour (OK, KO)" },
        { "name": "message", "type": "String", "description": "Message descriptif" },
        { "name": "reference", "type": "String", "description": "Reference de la commande" }
      ]
    },
    {
      "operation": "consult_commande",
      "method": "GET",
      "path": "/command-chequier/consult/{reference}",
      "summary": "Consulter le statut d'une commande",
      "idempotent": false,
      "timeout_seconds": 15,
      "max_retries": 2,
      "request_fields": [
        { "name": "reference", "type": "String", "required": true, "description": "Reference de la commande" }
      ],
      "response_fields": [
        { "name": "reference", "type": "String", "description": "Reference de la commande" },
        { "name": "statut", "type": "String", "description": "Statut (EN_COURS, LIVRE, ANNULE)" },
        { "name": "dateCreation", "type": "LocalDate", "description": "Date de creation" },
        { "name": "dateLivraison", "type": "LocalDate", "description": "Date de livraison prevue" }
      ]
    }
  ]
}

Fonctionnalites de resilience generees

Fonctionnalite Implementation Configuration
Circuit Breaker Resilience4j @CircuitBreaker Seuil 50%, 10 appels minimum, 60s demi-ouvert
Retry Resilience4j @Retry Configurable par endpoint (max_retries), backoff exponentiel
Bulkhead Resilience4j @Bulkhead 25 appels concurrents max, 5s attente
Idempotency IdempotencyService (ConcurrentHashMap + TTL) Base sur request_id, TTL 24h
Timeout RestTemplate connectTimeout/readTimeout Configurable par endpoint (timeout_seconds)
