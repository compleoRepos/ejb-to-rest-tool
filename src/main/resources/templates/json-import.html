Pipeline principal : Ce mode utilise exactement le meme moteur de generation que le mode ZIP/EJB. Le RestAdapter genere utilisera les URLs et methodes HTTP reelles du contrat JSON (GET/POST/PUT/DELETE/PATCH).
Pipeline JSON → BIAN ACL (Pipeline Principal)
Uploadez le contrat JSON normalise d'un Adapter. L'outil genere automatiquement via le pipeline principal :
  • Controller BIAN avec mapping Service Domain / Behavior Qualifier
  • Service Interface (couche domaine decouple)
  • RestAdapter avec la bonne methode HTTP (GET/POST/PUT/DELETE/PATCH)
  • DTOs Request/Response avec validation Bean Validation
  • Tests Pact + Integration
  • Configuration Spring Boot + Resilience4j + Actuator
  • Amelioration IA (SmartCodeEnhancer)
